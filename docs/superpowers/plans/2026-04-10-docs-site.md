# Docs Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a complete documentation site with a page for every component, standalone code examples, API reference, and changelog.

**Architecture:** Astro + Starlight docs site at `apps/docs/`. Each component gets an MDX page with: description, installation, import, usage examples (code blocks), props table, and related components. Sidebar organized into categories. No live previews (Storybook handles that).

**Tech Stack:** Astro, @astrojs/starlight, MDX

---

## Sidebar Categories

Components will be organized into these sidebar groups:

- **Getting Started** — installation, theming
- **Layout** — aspect-ratio, card, resizable, scroll-area, separator, skeleton
- **Forms** — autocomplete, button, button-group, calendar, checkbox, color-picker, combobox, command, date-selector, filters, form, input, input-group, input-otp, label, native-select, number-field, radio-group, select, slider, sortable, switch, textarea, toggle, toggle-group
- **Data Display** — accordion, alert, avatar, badge, breadcrumb, carousel, code-block, copy-button, data-grid, empty-state, json-viewer, kbd, pagination, progress, spinner, stat-card, status-indicator, table, timeline, tree
- **Feedback** — alert-dialog, dialog, drawer, dropdown-menu, hover-card, popover, sheet, sonner, tooltip
- **Navigation** — menubar, navigation-menu, sidebar, stepper, tabs
- **Charts** — EChart, BarChart, LineChart, AreaChart, PieChart, RadarChart, ScatterChart, chartConfig
- **Utilities** — useFileUpload, useMobile, cn
- **Changelog**
- **API Reference**

## MDX Template (per component)

Each component page follows this structure:

```mdx
---
title: ComponentName
description: One-line description.
---

## Installation

\`\`\`bash
npm install @e412/rnui-react @e412/rnui-themes
\`\`\`

## Import

\`\`\`tsx
import { ComponentName } from '@e412/rnui-react'
\`\`\`

## Usage

\`\`\`tsx
<ComponentName>Hello</ComponentName>
\`\`\`

## Examples

### Variant Name

\`\`\`tsx
<ComponentName variant="secondary">Secondary</ComponentName>
\`\`\`

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "secondary"` | `"default"` | Visual style |
| `size` | `"sm" \| "default" \| "lg"` | `"default"` | Size |
| `disabled` | `boolean` | `false` | Disabled state |

## Related

- [Button Group](/components/button-group)
```
