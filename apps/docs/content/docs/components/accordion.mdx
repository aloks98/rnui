---
title: Accordion
description: Expandable content sections that allow users to show and hide related content. Built on Base UI Accordion primitive.
---

## Installation

```bash
npm install @e412/rnui-react @e412/rnui-themes
```

## Import

```tsx
import {
  Accordion,
  AccordionItem,
  AccordionTrigger,
  AccordionContent,
} from '@e412/rnui-react'
```

## Usage

```tsx
<Accordion>
  <AccordionItem>
    <AccordionTrigger>Is it accessible?</AccordionTrigger>
    <AccordionContent>
      Yes. It adheres to the WAI-ARIA Accordion pattern.
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

## Examples

### Multiple Items

```tsx
<Accordion>
  <AccordionItem>
    <AccordionTrigger>What is this component?</AccordionTrigger>
    <AccordionContent>
      A vertically stacked set of interactive headings that each reveal a section of content.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem>
    <AccordionTrigger>Is it styled?</AccordionTrigger>
    <AccordionContent>
      Yes. It comes with default styles that match the design system.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem>
    <AccordionTrigger>Is it animated?</AccordionTrigger>
    <AccordionContent>
      Yes. It uses CSS animations for smooth open and close transitions.
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

### Open Multiple

Allow multiple items to be open at the same time by setting `openMultiple`.

```tsx
<Accordion openMultiple>
  <AccordionItem>
    <AccordionTrigger>Section One</AccordionTrigger>
    <AccordionContent>Content for section one.</AccordionContent>
  </AccordionItem>
  <AccordionItem>
    <AccordionTrigger>Section Two</AccordionTrigger>
    <AccordionContent>Content for section two.</AccordionContent>
  </AccordionItem>
</Accordion>
```

### Default Open

Set the default open item with `defaultValue`.

```tsx
<Accordion defaultValue={[0]}>
  <AccordionItem>
    <AccordionTrigger>Open by default</AccordionTrigger>
    <AccordionContent>
      This section is expanded on first render.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem>
    <AccordionTrigger>Collapsed by default</AccordionTrigger>
    <AccordionContent>
      This section starts collapsed.
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

### Disabled Item

```tsx
<Accordion>
  <AccordionItem>
    <AccordionTrigger>Enabled item</AccordionTrigger>
    <AccordionContent>This works normally.</AccordionContent>
  </AccordionItem>
  <AccordionItem disabled>
    <AccordionTrigger>Disabled item</AccordionTrigger>
    <AccordionContent>This cannot be toggled.</AccordionContent>
  </AccordionItem>
</Accordion>
```

## Props

### Accordion

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `openMultiple` | `boolean` | `false` | Allow multiple items open at once |
| `defaultValue` | `number[]` | — | Indices of items open by default |
| `value` | `number[]` | — | Controlled open items |
| `onValueChange` | `(value: number[]) => void` | — | Callback when open items change |
| `className` | `string` | — | Additional CSS classes |

### AccordionItem

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `disabled` | `boolean` | `false` | Disable the accordion item |
| `className` | `string` | — | Additional CSS classes |

### AccordionTrigger

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | Trigger label content |
| `className` | `string` | — | Additional CSS classes |

### AccordionContent

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | Expandable content |
| `className` | `string` | — | Additional CSS classes |
