## How It Works

The theme system uses **CSS custom properties** (variables) with **oklch** color values. Two packages work together:

- **@e412/rnui-themes** — defines all CSS variables and a Tailwind v4 `@theme inline` block
- **@e412/rnui-react** — components reference these variables via Tailwind utility classes

## Available Tokens

### Colors

| Token | Description |
|-------|-------------|
| `--background` / `--foreground` | Page background and text |
| `--card` / `--card-foreground` | Card surfaces |
| `--popover` / `--popover-foreground` | Popover/dropdown surfaces |
| `--primary` / `--primary-foreground` | Primary action color |
| `--secondary` / `--secondary-foreground` | Secondary action color |
| `--muted` / `--muted-foreground` | Muted/disabled elements |
| `--accent` / `--accent-foreground` | Accent highlights |
| `--destructive` / `--destructive-foreground` | Destructive/danger actions |
| `--border` | Default border color |
| `--input` | Input border color |
| `--ring` | Focus ring color |
| `--chart-1` through `--chart-5` | Chart color palette |
| `--sidebar-*` | Sidebar-specific variants |

### Border Radius

| Token | Value |
|-------|-------|
| `--radius` | Base radius (`0.625rem`) |
| `--radius-sm` | `calc(var(--radius) - 4px)` |
| `--radius-md` | `calc(var(--radius) - 2px)` |
| `--radius-lg` | `var(--radius)` |
| `--radius-xl` | `calc(var(--radius) + 4px)` |

## Light and Dark Mode

The light theme is applied on `:root`. The dark theme activates when the `.dark` class is on the `<html>` element:

```css
/* Light — active by default */
:root {
  --background: 1 0 0;
  --foreground: 0.145 0 0;
}

/* Dark — activated by .dark class */
.dark {
  --background: 0.145 0 0;
  --foreground: 0.985 0 0;
}
```

## Custom Themes

Create a custom theme by overriding the CSS variables under a custom class:

```css
.theme-ocean {
  --background: 0.97 0.01 230;
  --foreground: 0.2 0.02 230;
  --primary: 0.55 0.2 250;
  --primary-foreground: 0.98 0 0;
  /* ... override other tokens as needed */
}
```

Apply it by adding the class to `<html>`:

```html
<html class="theme-ocean">
```

## Tailwind v4 Integration

The `@theme inline` block in `@e412/rnui-themes` maps all CSS variables to Tailwind utilities:

```css
@theme inline {
  --color-primary: oklch(var(--primary));
  --color-background: oklch(var(--background));
  /* ... all tokens mapped */
}
```

This means `bg-primary`, `text-foreground`, `border-border`, etc. all work out of the box.
