## How It Works

`@e412/rnui-themes` defines design tokens as CSS custom properties using the [OKLch color space](https://developer.mozilla.org/en-US/docs/Web/CSS/color_value/oklch) — a perceptually uniform color model that produces more vibrant and consistent palettes than HSL.

Tokens are registered with Tailwind v4 via `@theme inline`, so you can use them as utility classes: `bg-primary`, `text-muted-foreground`, `border-border`, etc.

## Quick Start

```css title="app.css"
@import 'tailwindcss';
@import '@e412/rnui-themes';
```

This gives you the default warm neutral theme with light and dark mode support.

### Using a Preset

Import `presets` and set the `data-theme` attribute:

```css title="app.css"
@import 'tailwindcss';
@import '@e412/rnui-themes';
@import '@e412/rnui-themes/presets';
```

```html
<html data-theme="ocean">
```

Dark mode works with the `.dark` class as usual:

```html
<html data-theme="ocean" class="dark">
```

### Loading Fonts

Theme presets define `--font-sans` and optionally `--font-heading` variables, but **don't load the fonts automatically**. Load them via Google Fonts, `next/font`, or self-hosting. The font URL for each theme is documented in its CSS source file.

For example, for the Ocean theme:

```html
<link href="https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@400;500;600;700;800&display=swap" rel="stylesheet" />
```

### Switching Themes at Runtime

Since themes are `data-theme` attributes, switching is one line of JS:

```ts
document.documentElement.dataset.theme = 'violet';
```

### Scoping Themes to a Container

You can apply a theme to any element, not just `<html>`:

```html
<div data-theme="oxide">
  <!-- This section uses the Oxide theme -->
</div>
<div data-theme="rose">
  <!-- This section uses the Rose theme -->
</div>
```

## Available Tokens

### Colors

| Token | Usage |
| --- | --- |
| `--background` / `--foreground` | Page background and default text |
| `--primary` / `--primary-foreground` | Primary actions and buttons |
| `--secondary` / `--secondary-foreground` | Secondary actions |
| `--destructive` / `--destructive-foreground` | Destructive/danger actions |
| `--success` / `--success-foreground` | Success states |
| `--info` / `--info-foreground` | Informational states |
| `--warning` / `--warning-foreground` | Warning states |
| `--muted` / `--muted-foreground` | Muted backgrounds and text |
| `--accent` / `--accent-foreground` | Accent highlights |
| `--border` | Default border color |
| `--input` | Input border color |
| `--ring` | Focus ring color |
| `--focus` / `--focus-foreground` | Focus indicator color |
| `--invert` / `--invert-foreground` | Inverted color pair |
| `--chart-1` through `--chart-5` | Chart palette |

### Typography

| Token | Usage |
| --- | --- |
| `--font-sans` | Body text font family |
| `--font-heading` | Heading font family (falls back to `--font-sans`) |

### Shape

| Token | Usage |
| --- | --- |
| `--radius` | Base border radius (default: `0.625rem`) |

### Sidebar

Sidebar-specific tokens: `--sidebar`, `--sidebar-foreground`, `--sidebar-primary`, `--sidebar-primary-foreground`, `--sidebar-accent`, `--sidebar-accent-foreground`, `--sidebar-border`, `--sidebar-ring`.

## Light and Dark Mode

Light is the default; dark activates when `.dark` is on the `<html>` element (or any ancestor of the themed container).

You can import the base layers individually:

```css
@import '@e412/rnui-themes/base';   /* Reset and base styles */
@import '@e412/rnui-themes/light';  /* Light theme variables on :root */
@import '@e412/rnui-themes/dark';   /* Dark theme variables on .dark */
```

Or all at once:

```css
@import '@e412/rnui-themes';        /* Everything */
```

## Creating a Custom Theme

### Override the Default

Override any CSS variable after importing the base theme:

```css title="app.css"
@import '@e412/rnui-themes';

:root {
  --primary: oklch(0.55 0.2 260);
  --primary-foreground: oklch(0.98 0 0);
  --radius: 0.75rem;
}

.dark {
  --primary: oklch(0.7 0.18 260);
  --primary-foreground: oklch(0.1 0 0);
}
```

### Register a Custom Preset

Create your own named theme that works with `data-theme`:

```css title="app.css"
@import '@e412/rnui-themes';
@import '@e412/rnui-themes/presets';

/* Custom theme — only override what you need */
[data-theme="brand"] {
  --font-sans: 'My Brand Font', sans-serif;
  --radius: 0.75rem;
  --primary: oklch(0.55 0.2 180);
  --primary-foreground: oklch(0.97 0.01 180);
  /* Anything not listed inherits from the :root defaults */
}

[data-theme="brand"].dark,
.dark [data-theme="brand"] {
  --primary: oklch(0.72 0.18 180);
  --primary-foreground: oklch(0.15 0.03 180);
}
```

```html
<html data-theme="brand">
```

### Extend an Existing Preset

Start from a preset and tweak specific values:

```css title="app.css"
@import '@e412/rnui-themes';
@import '@e412/rnui-themes/presets';

/* Start from Ocean, customize primary */
[data-theme="ocean"] {
  --primary: oklch(0.48 0.15 220);
}
```

CSS specificity handles the cascade naturally — your override wins because it appears after the preset import.

## Using Tokens in Your Code

Since tokens are Tailwind classes, use them anywhere:

```tsx
<div className="bg-primary text-primary-foreground rounded-lg p-4">
  Custom themed box
</div>
```

Or access CSS variables directly:

```css
.my-element {
  background: var(--primary);
  color: var(--primary-foreground);
  font-family: var(--font-heading, var(--font-sans));
}
```
